Importing

Bring a module into scope with import std.<name>; an alias is optional:

import std.core.vec
import std.text.regex as re      // then call re.matches(...)

The compiler resolves std.sys.time to <LS_HOME>/lib/std/time.ls. std.core.str is auto-imported by the prelude, so string literals work without an explicit import.

Notation used below

  • &self = read-only borrow of the receiver; &!self = mutable borrow mutates.
  • &T / &!T = borrowed parameter (no copy, caller keeps ownership).
  • T value by value = the argument is moved in for move-types (Str, Vec, Map, structs/enums with a destructor), copied for POD.
  • name? ends a predicate (returns bool); name! is an unsafe / unchecked variant unsafe — you guarantee the precondition.
  • def map(U)(…) = a method-level generic; pass the result type explicitly: v.map(int)(|x| …).

Variable declarations

LS uses C-style type-prefixed declarations (no let); semicolons are optional:

Vec(int) v = [1, 2, 3]
Str name = "hello"
Map(Str, int) counts = {}     // {} = empty
Ownership in one line. Move-types die on rebinding (b = a moves a); passing them by value to a function clones; passing &a / &!a borrows. Containers and strings are destroyed automatically at scope exit. See ownership.md.

Imports & name resolution

How import finds a module, what the prelude gives you for free, and the exact rules for naming a type vs calling a function — bare, by alias, or by full path.

Import path = directory path

The standard library lives under <LS_HOME>/lib/std/, organised into category sub-directories. An import path maps dot-for-slash onto that tree:

import std.core.vec       // → lib/std/core/vec.ls
import std.sys.io         // → lib/std/sys/io.ls
import std.sys.c as c    // → lib/std/sys/c.ls

The seven categories: core (str·vec·map·hash·stack·num), mem (arena), sync (task·thread·atomic·lock·chan·ring), sci (complex·tensor·fft·rand), text (strconv·regex·json·html·md·bytes), sys (os·io·fs·path·env·proc·time·perf·c), chart (plot·plotfmt·plottl). A bare short name like import io does not resolve — always use the category path.

The prelude

std.core.str is the only auto-imported module: the compiler injects import std.core.str at the top of every root program (never into library modules). So the Str type and string literals work with no import:

def main() {
    Str s = "hello"      // no import needed — prelude
    @print(s.len())
}

Everything else is imported explicitly. Container types like Vec / Map frequently resolve without their own import too — the prelude pulls them in transitively through str — but write an explicit import std.core.vec for clarity.

Naming a type vs calling a function

This is the rule that catches people. A type name resolves through a global registry, so the bare name is the normal way to use it. A module function has no global name — it must be qualified by the module's namespace.

The one asymmetry: a bare function call never works — there are no global function builtins — whereas a bare type name always does.

The as rule for function namespaces

A plain dotted import binds only the full path, not the short namespace. To call strconv.fn() you must alias the import:

import std.text.strconv
Str h = strconv.int_to_hex(255)            // ✗ undefined variable 'strconv'
Str h = std.text.strconv.int_to_hex(255)  // ✓ full path works

import std.text.strconv as strconv     // ✓ now the short namespace is bound
Str h = strconv.int_to_hex(255)            // ✓

An alias binds the name for both roles: import std.text.json as j gives you the function namespace (j.parse(...)) and a type qualifier (j.JsonValue).

Special cases

  • No bare heap builtins. malloc / realloc / free / abort were removed as globals — a bare malloc(16) is undefined. Reach them through std.sys.c.
  • The four C primitives are ambient by full path. Those same four (std.sys.c.malloc/realloc/free/abort) resolve by full path with no import at all — the compiler special-cases the spelling. Any other std.sys.c function (e.g. c.fopen) still needs the import. This is what lets generic library bodies allocate without seeing the caller's import.
  • Qualified type names disambiguate. mod.Type / std.cat.mod.Type matter when two imported modules export the same type name — a bare Value would be ambiguous, so write mod_a.Value. With a single source, the bare type name is enough.
  • Cross-module privacy. Imported modules are namespaced; a module's free functions are mangled per module, so same-named functions in two modules never collide.

Error handling — Option / Result

Built-in tagged-union types for absence and fallibility, plus the operators and combinators that consume them. These are compiler built-ins (no import needed).

enum Option(T) { None, Some(T) }
enum Result(T, E) { Ok(T), Err(E) }

Most fallible stdlib functions return Result(T, Str) (the Err payload is a diagnostic message), and most container lookups return Option(T). Consume them with match, the try operator, the ! force-unwrap, or a combinator.

Operators

try expr Zig-style early return: unwrap Ok/Some, or propagate the Err/None out of the current function. Requires the function to return a compatible Result/Option.
expr! Force-unwrap: take the success value, or panic ([unwrap] line:col) and exit on None/Err. Bare variable form is (x)!.

Combinators (compiler-lowered, zero-cost)

.unwrap()Take the value or panic.
.expect(Str msg)Take the value or panic with msg.
.unwrap_or(T fallback)Take the value, or return fallback.
.unwrap_or_else(|| body)Take the value, or compute the fallback (closure; |e| for Result).
.is_some?() / .is_none?()Option predicates (borrow, don't consume).
.is_ok?() / .is_err?()Result predicates.
.map(U)(|x| body)Transform the success value → Option(U)/Result(U,E).
.and_then(U)(|x| body)Chain: body itself yields an Option(U)/Result(U,E).
.map_err(F)(|e| body)Transform the Err payload → Result(T,F).
res.ok() / res.err()Result(T,E) → Option(T) (discard err) / Option(E) (keep err).
opt.ok_or(E e)Option(T) → Result(T, typeof(e)), attaching e on None.
// match — the explicit form
match v.get(3) {
    Some(x) => { @print(x) }
    None    => { @print("out of range") }
}

// combinators — concise
int n = v.get(3).unwrap_or(0)
Str name = cfg.get("name").map_err(Str)(|e| f"missing: {e}").unwrap()

// try — propagate the error upward
def load(Str p) -> Result(Str, Str) {
    Str raw = try io.read_file(p)   // returns Err early on failure
    return Ok(raw.trim())
}

struct Vec(T) { *T data; int len; int cap }   // has destructor

ringimport std.sync.ring

Bounded FIFO ring buffer (rte_ring-style: power-of-2 sizing, idx & mask, free-running Atomic(i64) cursors). Lock-free across threads — no mutex. Pick the mode at construction.

struct Ring(T) { *T data; int cap; int mask; bool mp; bool mc;
                 Atomic(i64) prod_head; Atomic(i64) prod_tail;
                 Atomic(i64) cons_head; Atomic(i64) cons_tail }   // has destructor
def new_ring(T)(int capacity) -> Ring(T)        // SPSC: 1 producer + 1 consumer (no CAS, cheapest)
def new_mpmc_ring(T)(int capacity) -> Ring(T)   // MPMC: many producers + many consumers (CAS reserve)
                                               // capacity rounded UP to next power of 2
SPSC vs MPMC. new_ring is single-producer/single-consumer — one thread enqueues, one dequeues (single-threaded use is the degenerate case). new_mpmc_ring allows any number of producers and consumers via CAS slot reservation. Non-blocking: enqueue returns false when full, dequeue returns None when empty (busy-poll or back off yourself). For blocking message passing use Chan(T) instead.

arenaimport std.mem.arena

Bump (region) allocators over raw malloc: pay 1 malloc (amortized) + N pointer-bumps + 1 free instead of N malloc/free. Two flavors — a typed pool and a heterogeneous byte region. POD-only by contract (a bulk reset/free runs NO per-element destructor, so an element owning a malloc'd buffer would leak).

Arena(T) — typed pool, index handles

A growable pool of one POD element type, addressed by stable int index handles (survive realloc — link nodes by storing handles, for graphs / trees / ECS). O(1) reset() reuses the block across rounds.

struct Arena(T) { *T data; int len; int cap }   // has a ~ destructor (frees its one block)
def reserve(&!self, int need)Pre-grow capacity (geometric); one malloc to need rounded up.
def alloc(&!self, T x) -> int where T: PodBump-allocate one element; returns its stable index handle. where T: Pod rejects heap-owning (non-POD) element types at compile time.
def get!(&self, int i) -> TUnchecked copy-out of the element at handle i (POD value copy).
def get(&self, int i) -> Option(T)Bounds-checked copy: Some(elem) or None.
def set!(&!self, int i, T x)Unchecked raw overwrite of slot i (POD — no destructor to run).
def reset(&!self)O(1) bulk reclaim, KEEP the buffer for reuse (per-frame pattern).
def len/cap(&self) -> int · def empty?(&self) -> boolQueries.

Region — heterogeneous byte region + string interning

One *u8 block; sub-allocate typed *T slices of MANY POD types sharing one lifetime (ASTs / mixed node graphs). FIXED capacity (hands out raw pointers → must NOT realloc). Also interns Str values with zero per-string malloc.

struct Region { *u8 base; i64 off; i64 cap }   // has a ~ destructor (frees its one block)
def reserve(&!self, i64 cap)Allocate the one fixed backing block (bytes). Call once before use.
def alloc_bytes(&!self, i64 size) -> *u8Carve size bytes, 8-byte aligned; cast it (r.alloc_bytes(sizeof(T)) as *T). Aborts on overflow.
def str(&!self, &Str src) -> StrIntern a copy of the WHOLE src into the region; returns a region-backed Str (cap == -2 sentinel).
def str_substr(&!self, &Str src, int start, int len) -> StrIntern a SLICE src[start, start+len) — zero malloc: one region bump + one memcpy straight from src (no temporary Str, unlike r.str(src.substr(a,b))). The parser token pattern. Lenient clamping (matches Str.substr).
def reset(&!self)O(1) rewind, keep the block. Raw pointers / region Strs handed out before reset now dangle (caller's contract).
def used/cap(&self) -> i64Bytes used / block capacity.

Region-backed Str (cap == -2): the destructor ~ is a no-op (region reclaims in bulk); clone/copy deep-copies to malloc (PROMOTES off the region → independent, safely outlives reset()); mutation (push_str/+) also promotes. Best for IMMUTABLE interned strings (tokens / AST keys).

⚠ Pitfall: a region Str dangles after reset() or when the region is destroyed (not compiler-checked). A region Str's data points into the region block. Once that block is rewound by reset() or freed by the region's ~ destructor, every region Str (and raw pointer) handed out before then dangles — reads stale / overwritten bytes (UB). Two ways to get bitten: (1) use-after-reset — just holding one across a reset() in the same scope; (2) move-escape — moving one out past the region's life (a move only transfers the {data, len, cap:-2} handle as-is, still pointed into the region). The escape valve is clone: .copy() (or any auto deep-copy) PROMOTES to a fresh malloc → an independent owned Str (cap>0) that survives. LS has no lifetime system, so the compiler does NOT catch this — it is contract-based unsafe, like a raw pointer. Rule: any region Str that must outlive a reset() or the region itself must be .copy()'d first. 
// (1) use-after-reset — dangles even without moving anywhere
Region r = {}
r.reserve(64)
Str tok = r.str_substr(src, 0, 5)
Str keep = tok.copy()              // copy BEFORE reset -> independent malloc
r.reset()                            // block rewound: tok now dangles, keep is fine
@print(tok)                           // ✗ DANGLING (stale / overwritten bytes)
@print(keep)                          // ✓ OK

// (2) move-escape — handle returned past the region's life
def first_token(&Str line) -> Str {
    Region r = {}
    r.reserve(256)
    return r.str_substr(line, 0, 4)  // move-out: result points into r
}                                    // r destroyed here -> block freed -> result DANGLES; return tok.copy() instead
(Note: many LS sites — named-variable bindings / returns / pushes — actually clone and thus promote, which can mask the bug; don't rely on it, just .copy() what must survive.)
// parser pattern: carve tokens straight out of the source, zero malloc
Region r = {}
r.reserve(1 << 16)
Str src = "hello world"
Str tok = r.str_substr(src, 0, 5)   // "hello", region-backed (cap -2)
Str keep = tok.copy()               // PROMOTES to malloc → survives reset
r.reset()                            // O(1); tok now dangles, keep is fine

// typed pool with stable handles (graphs/trees/ECS)
Arena(int) a = {}
int h = a.alloc(42)
@print(a.get!(h))                     // 42
a.reset()                            // O(1) reuse

⚠ UNSAFE BY CONTRACT — a StrSlice must not outlive its source (compiler-unchecked). ptr points into a source Str's buffer; it has no ownership and extends nothing. conv1d is the SRS-CNN Main_conv block: a conv is a GEMM after im2col, so it reuses the register-blocked sgemm kernel. col is caller-provided (carve it from a Pool) so conv1d allocates nothing on the hot path. Wout = W + 2·pad − K + 1 (stride 1). The *_direct variants skip im2col entirely (read the padded input directly, register-blocked over output channels) — measured (SDE) win on small feature maps; im2col still edges them at large Wout≥16. Full NWc16 channel-blocked layout (vectorize channels) is a further layout deferred.
sgemm blocks M as 12→6→1 and N as 32→16→scalar: the 12×32 kernel (24 zmm accumulators — the BLIS-family AVX-512 server tile) and 12×16 (12 acc) hold C-rows in registers across K (one B-load feeds 24/12 FMAs — fewer loads, and ≥8 accumulators hide the FMA-recurrence on a 2-FMA-port core), falling to 6×16 and a 1×16 M-tail; the N tail is a scalar block. Measured (llvm-mca GNR + SDE): the wide kernels keep the FMA count and cut B-loads. BLIS A/B packing + cache blocking and the full NWc16 conv layout are not wired into the hot path: llvm-mca shows these kernels are FMA-port-bound (load ports have headroom), so packing optimizes a non-bottleneck on in-cache shapes — pure overhead here; its cache benefit only appears on out-of-L2 / many-channel workloads and is unmeasurable without real GNR. AMX-FP16 (tile engine, f32-accumulate) is a separate reference kernel — see model/amx/amx_sgemm_fp16.c + the model manual.

lock — Mutex / Guard / RwLockimport std.sync.lock

Bare locks (Mutex, SpinLock) and data guards (Guard, RwLock, SpinGuard) that make unlocked access a compile error.

How guards turn data races into compile errors — without lifetimes. A guard's value is priv (unreachable except through a closure), and the closure receives &!value as an arg-only, non-escaping borrow — it can't be returned, stored, or captured out. So the borrow provably lives only inside the critical section.
