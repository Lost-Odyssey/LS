print vec的元素时, 比如一个 vec(string) parts
print(parts[0]) 时 会发生 str.clone 然后在print之后 release
目前只要出现了 vec[0] 的访问, 一定会做深拷贝吗?


fn main() -> int {
    string csv = "apple,banana,cherry"
    vec(string) parts = csv.split(",")
    print(parts.length)   // 3
    print(parts[0])       // apple
    print(parts[1])       // banana
    print(parts[2])       // cherry
    
    return 0
}
